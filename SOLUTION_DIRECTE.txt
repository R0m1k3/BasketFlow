Le problème : PostgreSQL skip init.sql car le volume existe déjà

═══════════════════════════════════════════════════════════════

SOLUTION 1 : RÉINITIALISER COMPLÈTEMENT (Recommandé)
───────────────────────────────────────────────────────────────

cd /chemin/vers/basket-flow

docker-compose down -v

docker volume rm basket_postgres_data

docker-compose up -d


Attendez 30 secondes, puis testez :
http://localhost:4000/login (admin/admin)

═══════════════════════════════════════════════════════════════

SOLUTION 2 : FORCER L'EXÉCUTION D'INIT.SQL
───────────────────────────────────────────────────────────────

Si vous ne voulez pas perdre des données :

cd /chemin/vers/basket-flow

docker cp init.sql basket_postgres:/tmp/init.sql

docker exec basket_postgres psql -U basketuser -d basketdb -p 4532 -f /tmp/init.sql

docker-compose restart backend


═══════════════════════════════════════════════════════════════

VÉRIFIER QUE ÇA MARCHE
───────────────────────────────────────────────────────────────

docker exec basket_postgres psql -U basketuser -d basketdb -p 4532 -c "SELECT username FROM \"User\""


Doit afficher :
 username 
----------
 admin

═══════════════════════════════════════════════════════════════

SI TOUJOURS DES ERREURS
───────────────────────────────────────────────────────────────

1. Transférez le nouveau backend/Dockerfile (avec OpenSSL)

2. Reconstruisez :
   docker-compose build --no-cache backend

3. Réinitialisez tout :
   docker-compose down -v
   docker volume rm basket_postgres_data
   docker-compose up -d
