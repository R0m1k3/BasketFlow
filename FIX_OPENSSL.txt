❌ PROBLÈME IDENTIFIÉ
═══════════════════════════════════════════════════════════════

Erreur Prisma :
  "Error loading shared library libssl.so.1.1"

Cause :
  Alpine Linux n'a pas OpenSSL installé par défaut
  Prisma a besoin d'OpenSSL pour fonctionner

───────────────────────────────────────────────────────────────

✅ FIX APPLIQUÉ
═══════════════════════════════════════════════════════════════

Ajouté dans backend/Dockerfile :

  RUN apk add --no-cache openssl openssl-dev

Cela installe OpenSSL dans l'image Alpine.

───────────────────────────────────────────────────────────────

🚀 SUR VOTRE SERVEUR
═══════════════════════════════════════════════════════════════

1. Transférez le nouveau backend/Dockerfile

2. Reconstruisez l'image :
   docker-compose build --no-cache backend

3. Redémarrez :
   docker-compose up -d

───────────────────────────────────────────────────────────────

Plus d'erreur OpenSSL !
